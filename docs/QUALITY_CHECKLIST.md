# Crimson Signal Quality Checklist v1

## Purpose

This checklist exists to ensure every published page meets the Crimson Signal standard.

The goal is consistency.

A page should never be published simply because it is complete.

It should only be published when it genuinely helps someone make a better technology decision.

If a page fails any critical requirement, revise it before publishing.

---

# Mission Check

## Does this page help someone make a better decision?

YES / NO

If the answer is no, stop.

Rewrite the page.

---

# Reader Check

After reading this page, can someone confidently answer the primary question?

YES / NO

Examples

- Should I evaluate SD-WAN?
- Do I need managed networking?
- Should I replace my POTS lines?
- How much downtime is costing my business?

The page should reduce uncertainty.

---

# Editorial Review

## Quick Answer

✓ Answers the search intent immediately

✓ 40 to 80 words

✓ No dictionary definitions

✓ No marketing language

---

## Business Problem

✓ Explains the business problem before the technology

✓ Uses real operational examples

✓ Focuses on outcomes instead of features

---

## Should You Consider It?

✓ Clearly identifies ideal situations

✓ Clearly identifies situations where the technology is unnecessary

✓ Helps readers self-qualify

---

## Reality Check

✓ Includes one honest statement that challenges assumptions

✓ Builds credibility

✓ Does not exaggerate benefits

---

## Alternatives

✓ Discusses reasonable alternatives

✓ Does not assume one technology is always the answer

✓ Explains tradeoffs

---

## Questions to Ask

✓ Includes practical buying questions

✓ Helps readers evaluate vendors

✓ Avoids generic checklist items

---

## Bottom Line

✓ Summarizes the decision

✓ Makes the recommendation easy to understand

✓ Clearly states when the technology is appropriate

---

# Writing Quality

✓ No em dashes

✓ No AI clichés

✓ No marketing buzzwords

✓ Short paragraphs

✓ Clear headings

✓ Plain English

✓ Executive reading level

✓ Every sentence adds value

---

# Forbidden Language

The page should not include:

- Best-in-class
- Industry-leading
- Enterprise-grade
- Powerful
- Robust
- Innovative
- Revolutionary
- Comprehensive
- Seamless
- Cutting-edge
- Leverage
- Utilize
- Facilitate

If these appear, rewrite the sentence.

---

# Originality

Ask:

Could this paragraph appear on a vendor website?

YES

Rewrite it.

NO

Keep it.

---

# Original Insight

Every flagship page should contribute something that help the reader think differently about the problem.

Ask:

✓ Does this page contain at least three original insights?

Examples:

- A unique business perspective
- An unexpected tradeoff
- A recurring pattern across public case studies
- A practical recommendation based on evidence
- A misconception the research disproves
- An executive takeaway that reframes the problem

If the page only summarizes existing information, it is not ready to publish.

The goal is not to rewrite research.

The goal is to help readers think differently about the problem.

___

# Practical Value

Does the page include:

✓ Decision Matrix

✓ Decision Box

✓ Worth Knowing

✓ Reality Check

✓ Ask Before You Buy

✓ Continue Your Research

✓ FAQs

Every flagship page should include every component.

# Evidence Review

Every recommendation should be supported by:

- Public case studies
- Industry research
- Regulatory guidance
- Operational experience
- Published documentation

Avoid unsupported claims.

---

# Executive Test

Imagine the reader is a CIO, COO, CFO, or VP of Operations.

Would they say:

"I understand this topic better now."

YES / NO

Would they bookmark it?

YES / NO

Would they send it to a coworker?

YES / NO

If any answer is no, improve the page.

---

# Crimson Signal Test

Ask these final questions.

Does the page:

✓ Teach instead of sell?

✓ Explain the business problem before the technology?

✓ Acknowledge tradeoffs?

✓ Discuss alternatives?

✓ Build trust?

✓ Reduce uncertainty?

✓ Encourage informed decisions?

✓ Sound like an independent advisor?

✓ Avoid vendor language?

✓ Provide practical value?

If every answer is yes, the page is ready to publish.

---

# Publishing Standard

A Crimson Signal page should be the page we would want to find if we were researching this topic ourselves.

Publishing more pages is not the goal.

Publishing the most useful page on the internet for that topic is.
