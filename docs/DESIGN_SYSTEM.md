# Crimson Signal Design System v1

## Purpose

Crimson Signal should feel like an independent technology research platform.

Not a telecom vendor.

Not a marketing website.

Not a blog.

Every page should reinforce credibility, clarity, and decision confidence.

The interface should reduce cognitive load and help visitors move through a research journey.

---

# Design Principles

## Clarity over decoration

Every component should help someone make a better decision.

Avoid decorative elements that do not improve understanding.

---

## Scan before read

Executives rarely read top to bottom.

Design every page to be understood while scanning.

Use:

- Short sections
- Cards
- Tables
- Lists
- Callouts
- Icons only when helpful

---

## Build trust visually

The site should feel calm.

Avoid loud colors.

Avoid excessive gradients.

Avoid oversized hero sections.

The design should communicate confidence without trying to impress.

# Crimson Signal Design Principle

Every screen should answer this question:

"What decision is this helping someone make?"

If a component does not improve understanding, remove it.

Clarity is the design language.
